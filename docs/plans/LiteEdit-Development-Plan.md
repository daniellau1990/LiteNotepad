## 一、紧急修复 (v0.1.2)

### 1. handleOpenFile 定义顺序 Bug

**问题**: `src/App.tsx:85` - useEffect 中引用 `handleOpenFile`，但该函数在 useEffect 之后才定义

**影响**: 外部修改检测弹窗无法正常触发重新加载

**修复方案**: 将 `handleOpenFile` 函数声明移至 `useEffect` 之前，或使用 `useCallback` 包裹

**验收标准**:
- [ ] 外部文件修改后弹窗正常显示（如果需要此功能）
- [ ] 选择"重新加载"能正确读取文件内容
- [ ] 选择"覆盖"能保存当前内容

---

### 2. 窗口关闭保存缺失

**问题**: `src/App.tsx:160-170` - 监听 `tauri://close-requested` 但未执行实际保存操作

**影响**: 用户关闭窗口时未保存的更改会丢失

**修复方案**:
1. 在 close-requested 事件中检查 `isDirty` 状态
2. 如有未保存更改，先调用 `saveCurrentFile`
3. 等待保存完成后再允许窗口关闭

**验收标准**:
- [ ] 关闭窗口时如有未保存更改，弹出确认对话框
- [ ] 选择"保存"能正确保存并关闭
- [ ] 选择"不保存"能直接关闭
- [ ] 选择"取消"能阻止关闭

---

### 3. Save As 路径丢失

**问题**: `src/components/MenuBar.tsx:21` - `handleSaveAs` 调用 `onSave()` 而未传递 `save()` 对话框返回的路径

**影响**: "另存为"功能实际保存到了当前文件路径

**修复方案**:
```typescript
const handleSaveAs = async () => {
  const path = await save();
  if (path) {
    onSave(path);  // 传递新路径
  }
};
```

**验收标准**:
- [ ] "另存为"能保存到用户选择的新路径
- [ ] 保存后当前文件路径更新为新路径
- [ ] 状态栏显示正确的文件名

## 三、格式化工具栏 (v0.2.0)

### 设计方案

在编辑器下方或侧边添加格式化工具栏：

```
[B] [I] [标题▼] [列表▼] [链接] [图片]
```

| 按钮 | 功能 | Markdown 语法 |
|------|------|---------------|
| **B** | 加粗 | `**选中文字**` |
| *I* | 斜体 | `*选中文字*` |
| H | 标题 | `# ` / `## ` / `### ` |
| • | 无序列表 | `- ` |
| 1. | 有序列表 | `1. ` |
| 🔗 | 链接 | `[选中文字](url)` |

### 验收标准

- [ ] 点击加粗按钮，选中文字被 `**` 包裹
- [ ] 点击斜体按钮，选中文字被 `*` 包裹
- [ ] 快捷键 Ctrl+B 加粗选中文字
- [ ] 快捷键 Ctrl+I 斜体选中文字
- [ ] 未选中文字时，点击按钮在光标位置插入语法
