# 状态栏更新指示器提案

## 概述
增强 LiteEdit 状态栏功能，提供更多编辑器状态和文件信息的实时反馈，改善用户体验。

## 问题分析
当前状态栏显示信息有限：
1. 列号固定为 "Col 1"，不支持动态光标位置
2. 编码固定显示 "UTF-8"，未实际检测文件编码
3. 缺少行结束符指示器
4. 缺少大文件模式指示器
5. 缺少自动保存状态反馈
6. 缺少文件监视状态（外部修改检测）

## 功能需求

### 1. 动态光标位置指示器
- 实时显示光标所在行和列
- 支持多选区（显示主选区信息）
- 格式化显示：`Ln 10, Col 25` 或 `10:25`

### 2. 文件编码检测与显示
- 自动检测文件编码（UTF-8, UTF-16LE, UTF-16BE, ASCII, ISO-8859-1等）
- 显示当前编码，点击可切换编码（需确认）
- 编码异常时显示警告图标

### 3. 行结束符指示器
- 检测文件行结束符类型（CRLF `\r\n`, LF `\n`, CR `\r`）
- 显示当前行结束符：`CRLF`, `LF`, `CR`
- 点击可切换行结束符（需确认）

### 4. 大文件模式指示器
- 文件 >5MB 时显示大文件模式状态
- 指示器：`Large File Mode` 或图标
- 显示性能优化状态（流式读取、简化高亮）

### 5. 自动保存状态指示器
- 显示自动保存状态：
  - `Saved` (已保存)
  - `Saving...` (正在保存)
  - `Unsaved` (未保存，有修改)
  - `Error` (保存失败)
- 显示上次保存时间（可选）

### 6. 文件监视状态指示器
- 显示文件外部修改检测状态：
  - `Synced` (同步)
  - `Modified Externally` (外部修改)
  - `Watching` (监视中)
- 提供解决冲突的快捷操作

## 技术设计

### 架构概览
```
┌─────────────────────────────────────────┐
│              StatusBar Component         │
├─────────────────────────────────────────┤
│ 左侧区域              │ 右侧区域          │
│ ├─ 文件信息          │ ├─ 光标位置       │
│ ├─ 脏状态指示器      │ ├─ 编码信息       │
│ ├─ 大文件模式        │ ├─ 行结束符       │
│ └─ 自动保存状态      │ └─ 文件监视状态   │
└─────────────────────────────────────────┘
```

### 组件接口更新
```typescript
interface StatusBarProps {
  // 现有属性
  filePath: string | null;
  isDirty: boolean;
  lineCount: number;
  charCount: number;
  
  // 新增属性
  cursorPosition?: { line: number; column: number };
  fileEncoding?: string;
  lineEnding?: 'CRLF' | 'LF' | 'CR';
  isLargeFile?: boolean;
  autoSaveState?: 'saved' | 'saving' | 'unsaved' | 'error';
  fileWatchState?: 'synced' | 'modified' | 'watching';
  lastSavedTime?: Date;
}
```

### 数据流设计
1. **光标位置**: Editor → App → StatusBar
2. **文件编码**: 文件读取时检测 → App → StatusBar
3. **行结束符**: 文件读取时检测 → App → StatusBar
4. **大文件模式**: 文件大小检测 → App → StatusBar
5. **自动保存状态**: useAutoSave Hook → App → StatusBar
6. **文件监视状态**: useFileWatch Hook → App → StatusBar

## 用户界面设计

### 状态栏布局方案
```
[README.txt ●] [~/projects/liteedit/README.txt]          [Ln 42, Col 15] [UTF-8] [LF] [Large] [Saved 2m ago]
├─ 文件名+脏状态   ├─ 文件路径（可悬停显示完整路径）    ├─ 光标位置    ├─ 编码 ├─ 行尾 ├─ 大文件 ├─ 自动保存
```

### 交互设计
1. **悬停提示**: 各指示器悬停显示详细信息
2. **点击操作**: 部分指示器可点击进行操作（如切换编码、行结束符）
3. **状态颜色**: 使用颜色区分状态（正常、警告、错误）
4. **动画效果**: 状态变化时使用微动画增强可发现性

## 技术挑战与解决方案

### 挑战1: 编码检测准确性
- **解决方案**: 使用多重检测算法（BOM检测、字符分布分析、常见模式匹配）
- **备选方案**: 集成成熟编码检测库（如 chardet）

### 挑战2: 跨平台文件监视
- **解决方案**: 使用 notify 库提供统一接口
- **容错机制**: 轮询备选方案（当原生监视不可用时）

### 挑战3: 性能优化
- **防抖处理**: 光标位置频繁更新使用防抖
- **懒加载**: 编码检测在空闲时进行
- **增量更新**: 文件监视使用增量哈希比较

### 挑战4: 状态同步
- **状态管理**: 使用 React Context 或 Zustand 管理全局状态
- **事件驱动**: 基于事件的状态更新，避免循环依赖
